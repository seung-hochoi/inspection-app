# Google Spreadsheet 구조 스냅샷 v1 (2026-03-31)

대상 시트
- `매핑`
- `inspection_data`
- `inspection_summary`
- `return_exchange_records`
- `검품 회송내역 (센터포함)`
- `검품 회송내역 (센터미포함)`
- `제외목록`
- `행사표`
- `사전예약`
- `이력관리`

주의
- 이 문서는 현재 `Code.gs`가 기대하는 구조와 운영 헤더 기준 스냅샷입니다.
- `이력관리`는 현재 `Code.gs`에서 직접 생성/관리하는 시트는 아니며, 운영 관리용 시트로 간주합니다.

## 1. 매핑

용도
- 대분류 매핑
- 협력사 축약명 매핑
- 센터 표준값 보관

기준 열
- A열: `소분류명`
- B열: `대분류`
- E열: `센터`
- F열 또는 인접 값 영역: 센터 표준값
- G열: `협력사`
- H열: `값`

현재 런타임 사용
- `소분류명 -> 대분류`
- `협력사 -> 값`

참고
- 센터 매핑 데이터는 현재 시트 구조 문서상 보관하지만, 최신 런타임 로직에서는 직접 사용하지 않습니다.

## 2. inspection_data

헤더
- `작성일시`
- `작업기준일또는CSV식별값`
- `상품코드`
- `상품명`
- `협력사명`
- `전체발주수량`
- `발주수량`
- `검품수량`
- `회송수량`
- `교환수량`
- `사진개수`

설명
- 검품 row 원본 저장 시트
- 같은 상품/협력사/작업기준일 기준으로 검품수량은 마지막 값으로 유지

## 3. inspection_summary

용도
- 대시보드 시트
- `A1:F6` 범위에 요약 값 생성

구조
- `A1:F1`: 1차 지표 제목
- `A2:F2`: 1차 지표 값
- `A3:F3`: 2차 지표 제목
- `A4:F4`: 2차 지표 값
- `A5:F5`: 3차 지표 제목
- `A6:F6`: 3차 지표 값

## 6. 검품 회송내역 (센터미포함)

헤더
- `대분류`
- `상품코드`
- `파트너사`
- `상품명`
- `단위`
- `입고량`
- `검품량`
- `검품률`
- `교환 회송 내용`
- `불량률`
- `교환량`
- `회송량`
- `처리형태`
- `검품담당`

설명
- `inspection_data`와 `return_exchange_records`를 SKU 기준으로 합쳐 생성
- `대분류`는 CSV의 소분류명과 `매핑` 시트 기준으로 생성
- `파트너사`는 `매핑` 시트의 `협력사 -> 값` exact match로 축약
- `대분류`는 연속 구간 병합
- 표 범위 테두리 적용
